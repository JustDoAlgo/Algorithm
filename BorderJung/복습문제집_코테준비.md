## Part 1: BFS/DFS — 다시 풀어볼 문제

### 1-1. 네트워크 ★★ (연결 요소)
- 프로그래머스 Lv.3
- 복습 포인트: 전역 visited 초기화, 연결 요소 = BFS/DFS 호출 횟수
- 주의: `fill`/`memset` 초기화 빼먹지 않기

### 1-2. 아이템 줍기 ★★★ (좌표 2배 + BFS)
- 프로그래머스 Lv.3
- 복습 포인트: 좌표 2배 확대가 왜 필요한지 설명할 수 있는가?
- 셀프 체크: draw 시 `<=`인 이유 (점 vs 칸), 거리 /2 하는 이유

### 1-3. 게임 맵 최단거리 ★ (기본 BFS)
- 프로그래머스 Lv.2
- 복습 포인트: 가중치 1 격자 BFS 기본 템플릿 손에 익히기

### 1-4. 단어 변환 ★★ (BFS + 문자열)
- 프로그래머스 Lv.3
- 복습 포인트: 문자열 1글자 차이 판별 + BFS 최단 변환 횟수

### 1-5. 여행경로 ★★ (DFS 백트래킹)
- 프로그래머스 Lv.3
- 복습 포인트: 간선(티켓) 단위 visited, bool 반환 백트래킹, 알파벳순 정렬

### 추천 신규 문제 (BFS/DFS)
- 백준 2206 벽 부수고 이동하기 (3차원 visited BFS) — Unsolved에 있음!
- 백준 1697 숨바꼭질 (BFS 응용) — Unsolved에 있음!
- 프로그래머스 Lv.3 '단속카메라' (그리디이지만 그래프 사고 연습)

## Part 2: DP — 다시 풀어볼 문제

### 2-1. 사칙연산 ★★★ (구간 DP) ⚠️ 가장 중요
- 프로그래머스 Lv.3
- 복습 포인트: 구간 DP 템플릿을 빈 종이에 쓸 수 있는가?
- 셀프 체크:
  - [ ] 3중 루프 (len → i → k) 순서 기억
  - [ ] `i <= n-len` 범위 실수 안 하는가
  - [ ] dp_max/dp_min 초기값 (-INF, +INF) 까먹지 않는가
  - [ ] 분할점 k 기준 왼쪽 [i,k] 오른쪽 [k+1,j] 안 헷갈리는가

### 2-2. N으로 표현 ★★ (집합 DP)
- 프로그래머스 Lv.3
- 복습 포인트: "횟수 → 가능한 값 집합" 발상 전환
- 셀프 체크: sequence를 j 루프 밖에서 처리, 나누기 0 체크

### 2-3. 등굣길 ★★ (격자 경로 수 DP)
- 프로그래머스 Lv.3
- 복습 포인트: 좌표 매핑 (m=열, n=행) 먼저 정리하고 시작
- 셀프 체크: 첫 행/열 puddle 이후 break, 지역 배열 `= {}` 초기화

### 2-4. 도둑질 ★★ (원형 DP)
- 프로그래머스 Lv.4
- 복습 포인트: 원형 → 직선 2번 (0~n-2, 1~n-1)

### 2-5. 정수 삼각형 ★ (기본 2D DP)
- 프로그래머스 Lv.3
- 5분 안에 풀 수 있으면 PASS

### 추천 신규 문제 (DP)
- 프로그래머스 Lv.3 '단어 퍼즐' (구간 DP 연습)
- 백준 11053 가장 긴 증가하는 부분 수열 (LIS 기본)
- 백준 12865 평범한 배낭 (배낭 DP 기본)

## Part 3: 구현 — 다시 풀어볼 문제

### 3-1. 퍼즐 조각 채우기 ★★★ ⚠️ 중요
- 프로그래머스 Lv.3
- 복습 포인트: 좌표 정규화 + 회전 + 매칭 전체 흐름
- 셀프 체크:
  - [ ] 정규화: min(x,y)를 빼서 원점으로
  - [ ] 회전: (x,y) → (-y,x) 후 다시 정규화
  - [ ] 매칭: j % originCnt로 같은 물리 블럭 식별
  - [ ] break 빠뜨리지 않기

### 3-2. 다리를 지나는 트럭 ★
- 프로그래머스 Lv.2
- 복습 포인트: 시간 점프 시뮬레이션 패턴

## Part 4: C++ 필수 암기

### 배열 초기화
```
memset(arr, 0, sizeof(arr))     — 0 초기화 (#include <string.h>)
memset(arr, -1, sizeof(arr))    — -1 초기화
fill(&a[0][0], &a[0][0]+N*M, v) — 임의 값 (#include <algorithm>)
int arr[N] = {};                — 지역 배열 0 초기화
⚠️ memset은 0, -1만! fill은 &arr[N][M] 쓰면 segfault!
```

### 자주 실수하는 것들
- 전역 배열 초기화 (프로그래머스 다중 테스트케이스)
- m/n 가로/세로 혼동 → 변수 매핑 먼저 적기
- visited 배열 크기 (1-indexed면 n+1)
- dp 초기값 (최댓값 구할 때 0이 아닌 -INF)
- 루프 범위 `<` vs `<=`

## Part 5: SQL 복습

### 기본개념.sql 하단 연습 문제 다시 풀기
기록해둔 연습 문제를 보지 않고 풀어볼 것:
1. DATE() 함수 — DATETIME vs DATE 비교 차이
2. DATEDIFF + 1 — 양 끝 포함 경계값
3. LEFT JOIN + CASE ON — 동적 할인율 매칭
4. RANK vs ROW_NUMBER — 동점 처리 차이
5. LIKE는 패턴 하나만! — LIKE ('%A%', '%B%') ❌
6. NOT IN + NULL 주의 — 서브쿼리에 IS NOT NULL 필수
7. DISTINCT — 1:N JOIN 후 중복 제거

### 당일 워밍업용 SQL (뇌풀기)
- **셀프 JOIN**: '대장균들의 자식의 수 구하기' — 셀프 LEFT JOIN + COUNT(C.ID)
- **서브쿼리**: '그룹별 조건에 맞는 식당 목록 출력하기' — RANK() 인라인 뷰 + WHERE RNK=1 + 바깥 JOIN

### 가장 많이 실수한 SQL 패턴
- SELECT 별칭은 WHERE에서 못 씀 (실행 순서) → 인라인 뷰 또는 식 반복
- SUBSTR 인덱스는 1부터 (C++의 0-based와 다름!)
