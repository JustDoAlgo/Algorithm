## 자료구조 / 알고리즘

### BFS (너비 우선 탐색)

#### 핵심 개념
- 최단 거리를 구할 때 사용하는 대표적 탐색 알고리즘
- 큐를 사용하며, 방문 체크 + 거리 배열 관리가 핵심
- 시간복잡도: O(V + E) (정점 + 간선)

#### 실수 포인트 (260402 게임 맵 최단거리)

- **인덱스 off-by-one**: `t[n][m]`이 아니라 `t[n-1][m-1]`이 목적지
  - `size()`는 길이를 반환하므로 마지막 인덱스는 `size()-1`
  - 배열 선언 크기와 실제 접근 인덱스를 혼동하지 말 것

- **시작 거리 초기화**: 문제가 "칸 수"를 요구하면 시작점도 1로 세야 함
  - `t[0][0] = 0` vs `t[0][0] = 1` → 문제 조건을 먼저 확인
  - "거리"인지 "칸 수"인지에 따라 시작값이 달라진다

- **도달 불가 시 반환값 누락**: BFS 함수에서 큐가 비었는데 목적지 미도달이면 반드시 `-1` 반환
  - 반환 타입이 `int`인 함수에서 `return`이 없으면 **정의되지 않은 동작(UB)**

#### 제출 전 체크리스트 (BFS)
1. 시작점 거리 초기화 값 확인 (0 vs 1)
2. 목적지 인덱스 확인 (n-1, m-1)
3. 도달 불가 경우 반환값 처리
4. 방문 체크 타이밍 (큐에 넣을 때 vs 뺄 때)
5. 전역 배열 사용 시 초기화 누락 여부

## 문자열

### substr, rfind (260323 한국이 그리울 땐 서버에 접속하지)
- `substr(pos, len)`: pos부터 len만큼 부분 문자열
- `rfind(str)`: 뒤에서부터 찾기
- 패턴 매칭 시 prefix/suffix 분리 후 각각 검증
