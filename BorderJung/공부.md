- 메모리 상 연속한 구간 잡아야 해서 할당에 제약
- Cache hit rate 높음

## LinkedList
- k번째 원소 확인/변경: O(k)
- 메모리 상 연속하지 않아 할당이 쉬움
- Cache hit rate 낮음

    - it = erase(it) 형태로 써야 안전하다 !!

**insert(it, value)**: 현재 iterator 왼쪽에 value 삽입

## Array와 LinkedList
둘 다 선형 자료구조 이다. 둘의 차이를 알아두자.
|                          | 배열  | 연결리스트 |
| k번째 원소 접근           | O(1)  | O(k)      |
| 임의 위치에 원소 추가/제거 | O(N)  | O(1)      |
| 메모리 상의 배치          | 연속  | 불연속     |
| Overhead                 | -    | O(N)       |

Overhead는 배열은 굳이 따지면 길이를 저장하겠지만
연결리스트는 전 또는 후 pointer를 계속 저장해야 해서
overhead가 발생한다.

- UnorderedMap: 정렬 x

3. 시간 복잡도
Search, Insert, Delete에 대해

- UnorderedMap: O(1) ~ O(N)

## 문제별 메서드/문법 기록

### [BOJ 17478] 재귀함수가뭔가요
- 헷갈린 문법:
```cpp
string underbar(x * 4, '_');
```
- 의미:
  - 길이가 `x * 4`인 문자열을 만들고, 모든 문자를 `'_'`로 채운다.
  - 즉, `"____"`를 `x`번 반복한 것과 같은 들여쓰기 효과를 만든다.
- 왜 필요했는가:
  - 재귀 깊이에 따라 같은 패턴의 접두 문자열(들여쓰기)을 빠르게 생성해야 하기 때문.
- 비교 메모:
  - `underbar * x` 형태는 C++ `string`에서 지원되지 않는다.
  - C++은 파이썬처럼 문자열 곱셈 문법이 없어서 위 생성자 방식으로 처리한다.

### 문자열 생성자 기본형 메모
- 자주 쓰는 형태:
```cpp
string s(count, ch);
```
- 예시:
```cpp
string stars(5, '*');   // "*****"
string zeros(3, '0');   // "000"
```

### getline() 사용법
- `cin >>`은 공백에서 끊기므로, 공백 포함 한 줄 입력은 `getline`을 사용한다.
```cpp
string a;
getline(cin, a);
```
- `cin >>`과 섞어 쓸 때는 버퍼에 남은 `\n`에 주의 (`cin.ignore()` 필요할 수 있음)

### string::find()
- 특정 부분 문자열을 찾는 메서드. 찾은 문자열의 첫 인덱스를 반환하고, 못 찾으면 `string::npos` 반환.
```cpp
str.find(subStr, pos); // pos 위치부터 subStr을 탐색
```
- 겹치지 않게 찾기: `pos += subStr.size()`
- 겹쳐서 찾기: `pos += 1`

### string::replace()
- 문자열의 특정 위치부터 지정 길이만큼을 다른 문자열로 치환한다.
```cpp
s.replace(pos, length, newStr);
// pos: 시작 위치, length: 치환할 길이, newStr: 대체 문자열
```
- `find()`와 조합하면 특정 패턴을 일괄 치환하는 패턴으로 활용 가능:
```cpp
while ((pos = s.find(pattern, pos)) != string::npos)
{
    s.replace(pos, pattern.size(), "#");
    pos += 1;
}
```

### string::substr()
- 문자열의 일부를 추출한다. 두 번째 인자는 **길이**이다 (끝 인덱스가 아님).
```cpp
s.substr(pos, len);   // pos부터 len개 문자 추출
s.substr(pos);         // pos부터 끝까지 (len 생략 가능)
```

### string::rfind()
- `find()`의 역방향 버전. 뒤에서부터 탐색하여 마지막으로 등장하는 위치를 반환한다.
```cpp
s.rfind(pattern);      // 뒤에서부터 pattern 탐색
```
- suffix 매칭에 유용: `s.rfind(suffix) + suffix.size() == s.size()`로 끝부분 일치를 확인할 수 있다.

### char* → string 변환
- 대입 또는 생성자로 바로 변환 가능:
```cpp
char* cstr = "hello";
string s1 = cstr;    // 대입
string s2(cstr);     // 생성자
```

## 260323 문자열 문제 정리

### 문제: BOJ 1543 문서 검색
- 핵심 개념: 문자열 탐색, 겹치지 않는 패턴 카운팅
- 풀이 흐름: `string::find()`로 패턴 위치를 찾고, 찾으면 패턴 길이만큼 건너뛰어 겹침 방지
- 내 실수/막힌 지점: `getline()` 사용법을 몰라서 공백 포함 입력 처리에서 막힘
- 교정 포인트: 공백 포함 입력은 `cin >>` 대신 `getline(cin, s)` 사용
- 사용한 메서드/자료구조/알고리즘: `string::find()`, `string::npos`
- 시간복잡도/공간복잡도: O(N*M), O(1)
- 다음에 재사용할 패턴: `find() + pos += size()` 겹치지 않는 탐색 루프

### 문제: BOJ 2941 크로아티아 알파벳
- 핵심 개념: 다중 패턴 매칭, 문자열 치환
- 풀이 흐름: 8개 크로아티아 알파벳 패턴을 순회하며 `find()`로 찾고, `replace()`로 한 글자(`#`)로 치환 → 최종 문자열 길이가 답
- 내 실수/막힌 지점: 처음에 visited 배열로 겹침을 관리하려 했으나, 체크와 마킹을 동시에 해서 버그 발생
- 교정 포인트: 복잡한 상태 관리 대신 치환으로 단순화하는 발상 전환
- 사용한 메서드/자료구조/알고리즘: `string::find()`, `string::replace()`
- 시간복잡도/공간복잡도: O(8*N) ≈ O(N), O(1)
- 다음에 재사용할 패턴: `find() + replace()` 패턴 치환 루프 — 다중 패턴을 제거/축소할 때 유용

### 문제: BOJ 9996 한국이 그리울 땐 서버에 접속하지
- 핵심 개념: 와일드카드 패턴 매칭 (prefix/suffix 분리)
- 풀이 흐름: `*` 위치를 `find()`로 찾아 prefix와 suffix로 분리 → `substr()`로 추출 → `find()`로 prefix 매칭, `rfind()`로 suffix 매칭
- 내 실수/막힌 지점: `substr`의 두 번째 인자가 길이인데 인덱스로 착각. prefix+suffix 길이보다 짧은 입력에서 겹침 발생
- 교정 포인트: `substr(pos, len)`의 len은 **길이**임을 기억. 비교 전 `s.size() < prefix.size() + suffix.size()` 길이 체크 필수
- 사용한 메서드/자료구조/알고리즘: `string::find()`, `string::rfind()`, `string::substr()`
- 시간복잡도/공간복잡도: O(N * (P+S)), O(1)
- 다음에 재사용할 패턴: 와일드카드 `*` 패턴은 prefix/suffix 분리 후 양끝 매칭 — 길이 체크를 먼저 하는 습관
